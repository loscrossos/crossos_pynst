Initial base prompt for Crossos_Pynst
---


i need a python code that uses only standard libraries (no extra modules to be installed) and works on mac, Linux and Windows. The requirements are as follows:

i want a single python file that installs and manages a python repository project, the virtual environment (venv) and all related repositories contained in it. 
The definition of what is to be done is contained in input files.


-Arguments-

The code accepts 2 mandatory mandatory arguments

Mandatory arguments:

-an input file [file.pynst.txt]: The input file is a plain text file, where each line contains: a command in upper case followed by parameters if needed (see commands section)
-a target directory, where the pynst instructions will be applied. This can be relative or absolute

By default the code works in "install" mode, which installs a python repository and updates git repositories where it finds it (defined by the commands).


---
[COMAND] [PARAMETER1] [PARAMETER2] [PARAMETER3] 
---

The following commands exist unless stated explicitly they all can exist multiple times.  These commands must be executed in the order they appear in the file:

PYTHON: 1 Parameter: [a version number e.g. "3.12"]. This is the version of python to be used from this point on. if the version is not available abort with an error message. On Windows, the expected way to launch a specific Python version is via the py launcher e.g. "py -3.12". On Linux/macOS "python3.12".
RFILTER: multiple words.  it applies also to cloned repos. A list of packages that will be filtered out from all requirements files found from this line on (installations before this line are not affected). once RFILTER is set it applies to all requirement files in the order processed. yes, the filter is case-insensitive. Example: if the parameters are "torch torch vision" then from this point on all requirements.txt files found will be copied into a temp file before importing them and all lined containing the word "torch" or "torchvision" will not be copied. Only filter out if the word is alone or followed by a whitespace or non alphabetic symbol (A-Z a-z and - or _) symbol. "torchaudio" should not be filtered.
REQFILE: 1 Parameter: [a file path or URL]. a requirements file to be installed into the virtual environment. This can be a local file or a file on the internet that will then be downloaded into a temporary local file and installed from there.
CLONEIT: 2 Parameters: [an URL] [a path suffix]. The URL is the git repository to clone and install. The path suffix is used to set the target location of the clone operation. This is added to the base directory. e.g. base directory is /temp and suffix is "test" then the repository is cloned under /temp/test/. After the clone the code checks if a "requirements.txt" is present and if it's present it is installed in the virtual environment.

For example 
CLONEIT target .
[target dir]/projectx/requirements.txt
For target "mytest" :
[target dir]/mytest/projecty/requirements.txt


DESKICO, DESKEXE, HOMEEXE or HOMEICO: create a starter shortcut on the desktop ofthe current user. parameters: a label consisting of one word or several in quotes(labels in  quotes are treated as one argument), followed by multiple parameters: line is intended as the start command to be used to make a start shortcut file. For the EXE variant the file will be written as a .sh file or a .bat file (use bat for windows and sh for linux/macos) named after the label and  where the command that follows is added to the created venv. Example: if the base install is /temp/test and parameters are 'STARTER "Comfy CPU" comfyUI/main.py --cpu' then the starter on linux is the venv python and the starter is called "Comfy CPU.sh" with the contents "/temp/test/.env_linux/bin/python /temp/test/ComfyUI/main.py --cpu". for the ICO variants use shortcut formats for each OS (e.g. lnk for windows)
REQSCAN: 1 Parameters: [a path suffix]. Path under the base directory to be searched in repair mode for directories that contain "requirements.txt" files (only 1 level deep) to install into the venv in case of repair mode. in REQSCAN we only scan one level below the given suffix (RFILTER applies here too).  e.g. base directory is /temp and suffix is "test" then the code searches all directories under /temp/test/ and checks if these contain a "requirements.txt" and installs it. Match could be /temp/test/repo2/requirements.txt and /temp/test/repox/requirements.txt.  if install fails we abort and output the name of the path of the file that failed



Example: 
---
CLONEIT https://github.com/comfyanonymous/ComfyUI .
SETVENV ComfyUI
DESKICO "ComfyUI CPU Mode" ComfyUI/main.py --cpu  --auto-launch
---




-target directory: the path to be used as base directory to install the repositories or where the location of the existing installation is. 

Optional Arguments:

The code accepts the following optional arguments:

--senso installs the project without altering existing code or deleting files.
--revenv, rebuilds and repairs the python venv defined in the input file.
--embedded : declares the intsallation to be a portable embedded python installation.
--nodesktop : this sets a variable called STARTER_NO_DESKTOP as True in the code
--dryrun : this causes all runs to happen in simulated mode only with no changes to data for test purposes. The code prints all actions but never actually change the filesystem or install anything.
--verbose: by default command output are suppressed (by setting "capture=True" or similar. This option enables command outputs.
--backup: is a venv is already existent it will be renamed to [originalname].bak. else it is deleted first.




-Code Behavior-



Upon running the script checks the following:
-that python in is installed in the version defined in the default variable and the variable defined in the input file if that variable exists
-that git is installed in the system
-if the mandatory parameters are provided and valid. The path is created if it does not exist.

The code provides colorized output with defined colors for main tasks and other color for progress of subtasks while per default surpassing other output.



The code creates a virtual environment under the base directory using the version defined in the PYTHON command or the default version: If no PYTHON command is present the code uses 3.13 as default. This version is defined at the top of the code as a global variable.
Then it reads sequentially the other commands and performs the associated tasks.




If the argument --embedded is provided and the code detects the folder python_embedded then we have a portable installation and the following apply:
-there should be a folder named "python_embedded" directly under the target dir.
-the python executable python from that folder is used. it is located directly in the folder.

we assume the folder structure is:
[base dir]/python_embedded/

Check that the above apply. Else abort with the message "this is not a portable installation".





If needed the code will build a virtual environment in the following way:
in install mode the venv is always created under the base directory unless stated otherwise by SETVENV.
the name of the venv is according to the OS and a new venv will be created:
for windows: .env_win
for linux: .env_lin
for macOS: .env_mac

If the argument --backup is provided the venv directory already exists it will be renamed by adding the suffix ".bak" so its not lost.
If the venv is embedded then all packages will be removed by extracting them with pip freeze and unsinstalling them. 

then the venv will be rebuilt like this:

1. the venv is checked to be empty by running pip freeze. if not abort and show a error message.
2. requirements files defined in REQFILE will be installed with pip in the venv by downloading them to a temp file and performing pip install -r temp.txt
3. requirements files from all folders under the defined REQSCAN folders will be searched and installed with pip in the venv. 
4. requirement filed from all GITCLON folders are installed
5. If any file gives an error abort with a message displaying which custom node is the cause of error.




VENV Repair:

in revenv mode the venv of an embedded project might be corrupted by faulty entries in the pip freeze list. the list looks like this:
torch
torchaudio
WARNING: Ignoring invalid distribution ~orchsd (D:\gh_test\ComfyUI_windows_portable7\python_embeded\Lib\site-packages)
WARNING: Ignoring invalid distribution ~orch (D:\gh_test\ComfyUI_windows_portable7\python_embeded\Lib\site-packages)
torchvision

the code will clean the WARNING entries. but also repair the errors if those are found.
On each WARNING entry the path will be used and all directories on that path that start with a tilde "~" shall be deleted
